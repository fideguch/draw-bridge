## 1. 新しい危険の視覚言語（DangerZone / Spike）— 実装契約

並行タスク A1 が構築中の新 **DangerZone** エンティティに対する設計側の契約。**新エンティティは conventions.md / ubiquitous_language.md に先に追加**（CLAUDE.md「New entity → conventions.md first」/ learnings T2）。

### 1.1 DangerZone（危険帯）

- **形**: 静的 axis-aligned 矩形。level JSON `dangerZones[]`、キーは `{ x, y, width, height, kind }`。`(x,y)` = 左下コーナー（goalFlag と同規約：top = y+height）。
- **kind**: `"zone"`（純ハッチ矩形）/ `"spike"`（上辺に上向きの歯）/ `"spikeDown"`（下辺から下向きの歯＝天井鍾乳）。
- **判定**: 車の `VehicleReferencePoint`（chassis AABB center）または車体が矩形（spike は歯の外形）に接触したら**即 fail**。既存 fail 判定に「危険帯接触死」を追加（Rock と同様に転倒/落下/timeout とは別の即死ルート）。
- **描画（ゲーム内＋アトラス共通の視覚言語）**: `theme.uiDanger 0xff3b30` の縁＋半透明赤ハッチ塗り（`rgba(255,59,48,0.16)`＋45°ハッチ）。`spike`/`spikeDown` は base 矩形＋三角の歯（`stress` 系の赤 `#ff3b30`、縁 `#b32820`）。**歯は三角シルエットで“棘に見える”こと**（塊は禁止）。
- **level JSON 常駐**: 位置・寸法・kind は level 固有値 → level JSON に持たせる（TuningConstants には置かない。conventions §4 準拠）。
- **決定論**: 静的なので determinism に影響なし。負の対照（車を危険帯へ突っ込ませて必ず fail）を regression 固定（learnings T4）。

### 1.2 Rock（既存 RockHazard / conventions 既定）

円 body。level JSON `rocks[]` = `{ x, y, radius, density?, initialVelocity?{x,y} }`。重力落下/転がり、地形・線・車と通常接触、車に達すると既存 tipOver/fall/timeout を誘発（新 fail ルール不要）。決定論 60Hz 固定、spawn/初速は level JSON。

### 1.3 UL 追加（実装者向け）

`DangerZone`（危険帯・接触即死の静的矩形）/ `Spike`（DangerZone の歯付き変種）を UL に追加。既存 `Rock`/`RockHazard` は据え置き。禁止語彙は現状維持。

## 2. グローバル設計規約

- **座標系**: world メートル、y-up。アトラスの projector は `sy = maxY - y`（`scripts/atlas/levelCard.ts` と同規約）、viewBox = `minX 0 W H`。
- **車の設計速度**: 発射後の車体水平速度 `v_car ≈ 4.5 m/s`（wheel 表面 7.2m/s、平地の chassis 実速 4-5m/s。上りで減速・下りで増速）。交差 ETA はこの近似で算出（機械 gate が厳密固定）。
- **重力**: `g = 10 m/s²`。高さ h からの自由落下 ETA `t = √(2h/g) = √(h/5)`、着弾速度 `v = √(2gh)`。
- **無支持スパン上限 ≈ 5.5m**（`segmentLength=0.8`, `breakForceFactor=10`）。長い被覆/張り渡しは中州（柱）で分割。
- **落石荷重で平線は破断、アーチ/三角は荷重分散で耐える**（Stickman 攻略と物理一致）→ 直線否定(AD)を「インク欠乏」でなく「落石荷重／幾何」で作る。
- **鋭角コーナーは車を射出**（~1.4m）→ 多曲がり正解は spline 平滑化（描画は安定後カーブで表現）。
- **閉ループ非採用** → hook/wrap（L9/L12）は**開曲線**厳守。
- **線はソリッド（spike/地形/DangerZone）からクリップ**される既存仕様と全面整合。

### 2.1 アンカー分類（欠陥5 — 押しズレ対策）

| 方式 | 定義 | 該当面 |
|------|------|--------|
| **両リム支持** | 両端が対向するリムに載る | L1,L2,L9,L10,L11,B2 |
| **多点支持（中州）** | 両リム＋中央柱/島に載る | L3,L5,B1,L14,B3 |
| **地形ランプ延長** | 地形の傾斜面に載り短い延長で離陸 | L6 |
| **棚アンカー壁** | 棚/縁の角に基部固定、運動体が壁を支持へ押す | L7,L13 |
| **天井ブラケット＋外側リム** | 高所の庇/シュートを天井ブラケット＋ルート外リムに掛ける（落石が線を支持へ押し下げる＝ズレない） | L4,L8 |
| **ledge＋下段棚（降下）** | 上端 ledge・下端棚の両端支持で竪穴を降りる | L12 |
| **複合多点** | 蛇行を複数の地形点で支持 | L15 |

**自由浮遊線（片端のみ／どこにも載らない線）はゼロ**。純 shield 面（L4/L7/L8/L13）は線が非走行なので**コインは走行地形（車の実軌跡）に沿って配置**（research §5.3 の翻訳型 B。coin source を解ストロークから ghost samples[] へ切替）。dual-role 面（線＝道）は現行どおり線上に配置。

## 4. 面別詳細設計（18面）

> 各面: recipe/出典 / role / シルエット（テキスト略図）/ 縦活用 / ハザード配置（DangerZone・スパイク・岩の spawn/初速/交差ETA）/ アンカー方式 / なぜ直線が不可 / 難度位置。座標は world m（y-up）。詳細な数値は atlas-design.html の `LEVELS[]` を正とする。

### L1 — はじめの一歩（road / tutorial）★1
- **出典**: 既存チュートリアル。**シルエット**: `▁▁ ⌣ ▁▁`（浅い1穴に懸垂橋）。枠 12×8.5m（縦/横0.71）。
- **縦活用**: 谷底 killY-4.5 を見せ、橋のたわみで縦の存在を導入。
- **ハザード**: なし。
- **アンカー**: 両リム (-1.6,0)/(1.6,0)。自由端なし。
- **直線**: チュートリアルにつき直線可。「離す＝走る」を教える最初の面。

### L2 — のぼり坂（road / climb）★1-2
- **出典**: 既存チュートリアル / Draw Climber 上り。**シルエット**: `▁ ╱ ▔`（低左→高右棚へ上るランプ）。枠 13×10m（0.77）。
- **縦活用**: ゴール棚を spawn より **+2.6m 上**に置き上方向を使う。killY-5。
- **ハザード**: なし。
- **アンカー**: 左リム (-1.4,0) と 右棚縁 (2.4,2.6) の両端支持。
- **直線**: 直線ランプが答え（角度で走破を学ぶ）。

### L3 — ふたつの谷（road+support）★2 · AD幾何
- **出典**: 既存 / Draw Line Bridge 中州。**シルエット**: `▁ ⌣∏⌣ ▁`（中央柱＋二連谷）。枠 14×9m（0.64）。
- **縦活用**: 中央柱を killY-5 から立て、縦の支持構造を提示。
- **ハザード**: なし。
- **アンカー**: 左リム＋中央柱天 (0,0.3)＋右リムの3点支持。
- **直線**: リム間6mを無支持直線→破断。**中央柱に載せる必要**（AD幾何）。

### L4 — 落石の庇（R04 shield-static）★3 · rock · AD落石
- **出典**: Stickman Rescue（傾け屋根）/ Happy Glass L20+。**シルエット**: 平地レーン＋高い落石シャフト＋斜めの庇＋右端の側溝。枠 11×11.3m（**縦/横1.03・縦＞横**）。
- **縦活用**: 落石シャフト y+7〜側溝 killY-3。
- **ハザード（rock）**: spawn `(0.3, 7)`、`vx=0`（自由落下）、radius 0.4。**交差ETA**: 落下 h≈6.6m → `t=√(6.6/5)≈1.15s`。車 spawn(-4.5) → x0.3 到達 `(0.3+4.5)/4.5≈1.07s`。**線なし＝落石が (0.3,0) の車を直撃**（相関性成立）。正解時は庇が落石を右へ転がし右端の側溝(killY)へ落とす。
- **アンカー**: 左=天井ブラケット (-2.1,2.8)、右=ルート外の柱 (2.4,2.2)。落石は庇を下へ押し込む＝支持へ押す＝車接触ゼロでズレない。
- **直線**: 線は道でなく「蓋」。道は地形（連続床）。**平橋は落石荷重で破断/貫通**。コインは走行地形側に配置。

### L5 — 三つの穴（R08 multi-seal）★3 · AD幾何
- **出典**: Draw Line Bridge / Happy Glass 少描画。**シルエット**: `▁▽∏▽∏▽▁`（櫛形＝2柱・3穴）。枠 14×8.5m（0.61）。
- **縦活用**: 3穴を killY-5.5 まで、柱2本を縦に。
- **ハザード**: なし。
- **アンカー**: 左リム＋柱1(-1.6)＋柱2(0.9)＋右リムの**4点支持**。
- **直線**: 総幅6.8mを無支持直線→破断。**2柱に載せ一筆で全穴被覆**。

### L6 — 跳べ（R10 ramp-jump）★3 · AD スパイク
- **出典**: Draw Climber / Draw the Hill（launch ramp）。**シルエット**: `╱` 上りランプ→**スパイク谷**を放物線→着地台。枠 15×10.5m（0.70）。
- **縦活用**: 放物線峰 y+2.6、スパイク谷 killY-5。
- **ハザード（DangerZone spike）**: `{x:-1.0, y:-4.6, w:5.5, h:0.9, kind:"spike"}`（谷底に上向き9歯）。
- **アンカー**: 地形ランプ (-1.0,1.0) に載る短い延長で離陸 (0.3,1.9)。地形が下支え＝押されない。
- **直線**: スパイク谷（幅5.5m）は走破・架橋不可。**ランプ角＋助走（離陸≈5.5-6m/s）で飛び越えるのみ**。

### L7 — 転石を止めろ（R05 shield-dynamic-block）★3-4 · rock · AD転石
- **出典**: Stickman Rescue（矢=壁）/ Draw Bridge cover。**シルエット**: 上段メサ（右）の岩＋縁の**縦ストップ壁**、下段に車レーン。枠 13×9m（0.69）。
- **縦活用**: 上段メサ y+4.6 に岩、下段 y0 に車。**上下二段**で縦を使う。
- **ハザード（rock）**: spawn `(5.6, 4.4)`（傾斜メサ top）、`vx=-1.0`、radius 0.38。メサ勾配≈23°で転がり下る。**交差ETA**: 縁 (1.2,2.5) へ ≈1.4s（rolling accel≈2.6m/s²）＝車が x1.1 直下に到達 `(1.2+5)/4.5≈1.38s` と一致。**壁なし＝縁から車レーンへ落下し直撃**。
- **アンカー**: 壁の基部を上段メサ縁 (1.2,2.5) に固定。転石は壁を棚の角へ押す＝自分の支持へ押す＝安定。車レーン外なので車も押さない。
- **直線**: 転がってくる岩は「落下」でなく「横移動」。**縁に縦の壁**を立てて堰き止める。道は下段地形。

### L8 — 逸らす滑り台（R06 catch-redirect）★4 · rock · AD転石
- **出典**: Draw Physics Line / Brain Dots（redirect）。**シルエット**: 天井ブラケットから**急斜シュート**で岩を右の側溝へ落とす。枠 11×10.7m（**縦/横0.97・ほぼ縦**）。
- **縦活用**: 岩 y+5.6 から、側溝 killY-4 へ。
- **ハザード（rock）**: spawn `(-2.5, 5.6)`、`vx=+3.5`（左上から放物）、radius 0.38。**交差ETA**: 無遮蔽なら y0 着地 `t=√(5.6/5)≈1.06s`、x=-2.5+3.5·1.06≈1.2 ＝車位置（車 x1.2 到達 ≈1.2s）。シュートが弧の途中（≈0.85s, ~(0.8,2.0)）で受けゴール奥の側溝(killY)へ排出。
- **アンカー**: 左=天井ブラケット (-0.5,3.4)、右=側溝リム (3.0,0.2)。岩がシュート上を滑り押し下げる＝支持へ押す＝安定。
- **直線**: 止めるでなく「受けて逸らす」。**急斜シュートで転石をゴール奥の側溝へ排出**。道は地形。（L4 との差: L4=水平寄りの庇で岩を地形リムへ / L8=急斜シュートで深穴へプランジ、岩は空中から弧入り。）

### L9 — 鍾乳の下をくぐる（R16 hook-cantilever wrap / 護り道）★3-4 · AD スパイク
- **出典**: Draw Bridge L210 / Draw Save!。**シルエット**: 天井から**下向きスパイク鍾乳石**、その下を谷型でくぐる（低左→高右）。枠 12×8.5m（0.71）。
- **縦活用**: 鍾乳スパイクを y0.6 まで垂らし、谷底 killY-4 でくぐる空間を確保。
- **ハザード（DangerZone spikeDown）**: `{x:-0.55, y:0.6, w:1.1, h:2.0, kind:"spikeDown"}`（天井ブラケットから下向き3歯、尖端 y0.6）。
- **アンカー**: 低左リム (-2.4,0.2)＋高右棚 (2.4,1.8) の両端支持。中央でスパイク尖端の下へ**谷型で回り込む**。

## 7. 検証（own-eyes / 欠陥1-5）

atlas-design.html を Playwright でレンダリングし自眼レビュー（harsh judge）：

- **欠陥1（縦活用）**: 18面すべて 縦/横 ≥0.60、うち **L4(1.03)/L8(0.97≈)/L10(1.00)/L12(1.16)** が縦≥横。縦は「空の深穴」でなく段層ゲームプレイ（降下・落石シャフト・二段メサ・棚上ゴール・棘峡谷・登坂）で占有。✅
- **欠陥2（危険可視化）**: DangerZone は赤ハッチ矩形、スパイクは三角の歯で描画（L6/L9/L11/L12/B2/L15）。塊なし。✅
- **欠陥3（異形）**: §5 マトリクスで全153ペア ≥2軸相違・混同0。旧 L9/L12 は 4軸相違。✅
- **欠陥4（岩を当てる）**: 全 rock 面（L4/L7/L8/L10/L13/L14/L15/B3）に spawn・初速・交差ETA を明記、線なし＝直撃を相関性として記述。並行 hazard-relevance gate が機械強制。✅
