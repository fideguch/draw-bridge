## 0. エグゼクティブサマリ（結論先出し）

1. **ユーザーの「初期インクが少なすぎて単純」は、コードで裏が取れる正しい指摘。** 現行の authoring パイプライン（`scripts/levels/authoring.ts` L169-171）は、**anti-dominant 面のインク予算を「解の生ローlength × 1.06」または「最小消費 × 1.08」に固定**している。つまり **正解の線とほぼ同じ長さしか描けない**＝「工夫の余地ゼロ・唯一解」。競合（Happy Glass / Brain Dots / Love Balls）は逆に **クリア予算を最小解の 2〜4 倍取り、"少なく描くほど星が高い" を効率スコア軸に回している**。**難度は「インク欠乏」ではなく「幾何・物理の空間制約」で作るのが正解。**

2. **推奨: anti-dominant のインク欠乏オーバーライドを撤廃し、クリア予算を全面 2.0〜3.0×（最小解基準）に引き上げる。** 直線否定は **100% 地形幾何（壁・天井・スパイク・登坂・深穴）＋ Gate 3** に移す。これで「様々な図形を描いて工夫する」創造的余地が生まれ、効率（短く賢く）は星の軸に純化する。ユーザー要望の 3 点（地形パターン増・空間脳トレ・難度インフレ）が一気に噛み合う。

   | inkFeel | 現行係数 | **v3 係数** | 意味（最小解ink基準の倍率） |
   |---|---|---|---|
   | generous（Tutorial/Bonus） | 2.6× | **3.0×** | 潤沢に描いても通る |
   | standard（中盤） | 2.0× | **2.5×** | 曲線を試行できる余白 |
   | tight（終盤/Boss） | 1.5× | **2.0×** | 精密が報われるが、なお創造可 |
   | **anti-dominant override** | **~1.06×** | **撤廃** | 直線否定は幾何へ |

   星閾値ポリシー（**最小解ink基準の倍率**で定義、Happy Glass「少ない=良い」方式）:
   **star3(金) ≤ min×1.10 / star2(銀) ≤ min×1.35 / star1(銅) = クリアのみ（≤ budget）**。
   → 例: standard 面は budget 2.5× / 銀 1.35× / 金 1.10×。**雑に描いても銅で通り、半分に削れば銀、最短で金**という効率ラダー。AD 面の実インクは **~1.06× → 2.0× へほぼ倍増**＝ユーザーの「少なすぎ」に直接回答。

3. **地形パターンは 13 型 → 39 採用型（+ 非採用 3 型）に拡張（本書 §2）。** 8 ファミリ（Gap 渡し / 支点 / 登坂 / 降下 / 天井障害 / 縦壁障害 / トラップ幾何 / 複合コリドー）に分類。各型に **強制ストローク形（Line/Hump∧/Scoop⌣/Ramp/S/W/Flat-hug/Cantilever/Wedge/M）・難度 tier・ポリライン実現可否＋注意** を付す。**全採用型がエンジン改修ゼロで実装可能**（複数静的ポリライン + winding 反転天井 + spike/pillar + goal 上下 + killY + coin のみ）。**18 スロット（Ch1）に 15 型を配し、残り 24 型は Ch2+ のストック**。

4. **本ゲームは「落下する球を器で受ける」型ではなく「車が描いた橋を走る」型。** ゆえに Happy Glass の "容器/漏斗/受け網" はそのままは移らない。**強制されるのは "駆動面の形"**（登る/降る/潜る/越える/谷を通す）。マンデート語彙（scoop/loop/wall/wedge/multi-bend/enclosure）は車橋文脈に翻訳して割当て、**enclosure/loop（閉曲線）は Box2D チェーンでの走行成立性が未検証なので Ch1 非採用（§2 で明示）**。

5. **難度インフレ v3（§4）は「1 面 1 パターン → 2 パターン複合 → 3+ 複合」で組合せ数を単調増加。** 直線が失敗する理由・強制ストローク形・線長・曲がり数・縦使用・インク感を 18 スロットで指定。**新機構の直後とボーナスで必ず難度を落とす鋸歯**（"saw curve" — 競合共通則）。ストローク: 長さ 4.5m→~16m、曲がり 0→4、縦使用 0.5m→6m。

| 制約 | 根拠 | パターン設計への含意 |

### 1.3 Gate 3 と「予算引き上げ」の相互作用（重要な実装注意）

現行 `gate3-antidominant.ts` は `if (length > level.inkBudget)` で **予算超過の overlapped 直線を `infeasible(budget)` として物理実行せず失敗扱い**にしている（L103-104）。**予算を 2.0× に上げると、overlap{1.0,2.0}m の直線が予算内に入り、実際に物理実行される**。よって:

- これらの overlapped 直線が **幾何で必ず落ちる（sag 破断 / 壁激突 / スパイク激突 / 登坂不能 / killY 落下）ことを各面で再検証必須**。既存 AD 面のゴーストコメント（"all 9 Gate-3 straights fail"）は現状 budget-infeasible に頼った面も含むため、**予算引き上げ後は物理での全滅を再確認する**。
- Gate 3 は **候補全てを 1 個の recycle world で回す**（`gate3-antidominant.ts` L15-17）ので、**32-slot cap は破らない**（予算オーバーライドは slot 対策としては不要だった）。増えるのは実行時間のみ。
- 万一どうしても物理で落とせない overlapped 直線が残る面だけ、**その面に限り budget を控えめ（例 1.8×）に据える** 局所調整を許可（撤廃は原則・例外は面単位）。

## 3. インク予算ポリシー（競合エビデンス + 推奨数値）

### 3.1 競合の共通処方: 「クリア予算 >> 最小解、効率は星の軸」

| ゲーム | クリアに必要なインク量 | 星（効率）の扱い | 予算/最小比の実態 |
|---|---|---|---|
| **Happy Glass**（Lion, 500 面） | **実質無制限**（好きなだけ描いて OK） | 「描いた量が少ないほど 1〜3 星（less is better）」。**クリア自体は最小描画を要求しない** | クリア ≈ ∞×min、星3 ≈ 最小近傍 |
| **Brain Dots**（TransLimit） | **「不格好な三角形の寄せ集め」でもクリア可**（＝大量に描ける） | 「線と図形の長さで評価、短いほど高評価」 | クリア ≈ 3〜4×min+、評価は短さ |
| **Love Balls**（SuperTapx） | 多めに描いても解ける | **3 星は "描いた線の本数/長さに上限"** | クリア >> 3 星閾値 |
| **Draw Bridge**（Bravestars, 最直接競合・1 本線） | 1 本、障害次第 | 難度は **壁/落とし穴/障害（幾何）** で作る（インク欠乏ではない） | 幾何ファースト |
| **Brain It On!**（Orbital） | 複数図形を自由に試せる | 目標達成が主、創造性優先 | 潤沢 |

**結論（3 点）:**
1. **どの競合も「クリア予算 ≈ 最小解の 2〜4 倍以上」を確保し、"少なく描くほど高星" を効率スコアに回す。** 難度は幾何/物理で作り、インク欠乏では作らない。
2. **InkBridge の現行 anti-dominant（~1.06×）は業界と真逆**＝唯一解に押し込め、創造も精密報酬も殺している。ユーザーの「単純」体感の機序はこれ。
3. **修正方針: クリア予算を上げ（創造の余白）、直線否定を幾何へ移し（Gate 3）、効率は星に純化する。**

### 3.2 推奨: base-budget multiplier

> `inkBudget = feelFactor × (最小解の実測消費 tightRefInk)`。`FEEL_FACTOR`（`authoring.ts` L55-59）を下表へ改定し、**anti-dominant の予算オーバーライド（L169-171）を撤廃**する。

| inkFeel | v3 係数 | 用途 | 備考 |
|---|---|---|---|
| `generous` | **3.0×** | Tutorial(L1-L2)・Bonus(B1-B3)・息抜き | 潤沢に描いても通す。失敗回避優先 |
| `standard` | **2.5×** | 中盤の主力(L3-L10) | 曲線を数回試せる余白。工夫が star へ |
| `tight` | **2.0×** | 終盤/Boss(L11-L15) | 精密が報われるが **床は 2.0×**（1.5× に戻さない） |
| anti-dominant | **（撤廃）** | — | 直線否定は §1.3 の幾何＋Gate 3 で担保。予算は tier の係数を使う |

- **床 = 2.0×。** ユーザー「初期インクが少なすぎる」への直接回答として、**どの面も最小解の 2 倍は描ける**ことを保証。AD 面の実インクは **~1.06× → 2.0×（ほぼ倍増）**。
- **上限（AD 面の Gate 3 保険）:** 予算が上がると overlapped 直線が物理実行される（§1.3）。**幾何で全滅を再検証**。どうしても落とせない稀な面のみ **1.8× まで局所的に絞る例外**を許可（原則撤廃・例外は面単位・コメント必須）。

### 3.3 推奨: star-threshold policy

> 星は **最小解 ink（星3ゴーストの実測消費 = tightRefInk）基準の倍率**で定義（Happy Glass 方式）。`authoring.ts` の非 AD 派生（現 star3=×1.05 / star2=×1.25）を微調整。

| 星 | 閾値（min ink 基準） | 意味 |
|---|---|---|
| ★★★ 金 | **≤ min × 1.10** | ほぼ最短。精密の到達点 |
| ★★☆ 銀 | **≤ min × 1.35** | 概ね効率的。無駄弧を削れば届く |
| ★☆☆ 銅 | **≤ inkBudget**（= 2.0〜3.0×min） | クリアのみ。雑でも通る（創造の受け皿） |

- **順序保証:** `0 < star3(1.10×) < star2(1.35×) < inkBudget(2.0〜3.0×)`。`validateLevel` の `star2 <= inkBudget` を常に満たす（現行 L416 と整合）。
- **体感:** 「不格好でも銅で通る → 弧を削れば銀 → 最短で金」。Brain Dots/Happy Glass と同じ効率ラダー。**銅↔銀の gap が広い**（1.35×→2.5×）ほど「無駄を削る」達成感が出る。
- **3star ゴースト:** AD 面も `kind:'3star'` ゴーストを 1 本添えれば、金閾値（1.10×）を Gate 2 が assert でき、効率軸が全面で機能する（現行は AD 面が 'any' のみ＝効率軸が死んでいた。§ の改善点）。

### 3.4 Ink Capacity アップグレードとの整合

- `effectiveInkBudget = level.inkBudget × (1 + 0.10 × inkCapacityLv)`（game_design §7、Lv5 で +50%）は**そのまま維持**。base を 2.0× 化しても「Lv0 で全面クリア可能」は保たれる（むしろ余裕増）。
- アップグレードの意味が「唯一解をなぞる」から「**より大胆・創造的な橋（吊り/二重支持）を組む自由度**」へ変わる＝game_design §7 の設計意図（「創造的解の解禁」）が初めて機能する。

## 5. 実装ノート・制約順守まとめ

1. **エンジン改修ゼロ。** 使う道具は既存の「複数静的ポリライン（壁/天井/柱/庇は winding/spike/pillar）」「goal 上下」「killY」「inkBudget/star」「anti-dominant + Gate 3」のみ（§1.1）。
2. **`authoring.ts` の 2 点改修が本書の実装本体:**
   - `FEEL_FACTOR` を `{generous:3.0, standard:2.5, tight:2.0}` に（L55-59）。
   - **anti-dominant 予算オーバーライド分岐（L169-171）を削除**し、AD 面も tier 係数で予算導出。star 派生は `star3=×1.10, star2=×1.35`（L168/173 を調整）。**AD 面に `kind:'3star'` ゴーストを 1 本追加**して効率軸を有効化。
3. **予算引き上げ後の Gate 3 再検証（§1.3 必須）:** overlapped 直線が物理で全滅することを面ごとに確認。落とせない稀な面のみ 1.8× 局所例外（コメント必須）。
4. **長線は「支持あり」で設計**（無支持 ~5.5m 上限）。長さは縦の蛇行/降下/登坂で稼ぐ（水平フットプリント ≤~8-11m）。
5. **多曲がり正解は spline 平滑化**（鋭角コーナーの車射出回避、`ch1.ts` `spline()`）。
6. **coin は正解軌道の弧状ヒント**（`coinArc`/`coinLine`）。
7. **新地形（トンネル/鍾乳石/二重壁/九十九折）はゴースト録画で破断/転倒/予算を実測校正**（learnings: ハンドチューニング禁止、spike sweep で確定、決定論で再現固定）。△ 判定の 7 型（A04,B05,D03,F03,G03 等）は特に校正コスト高。
8. **非採用を明示遵守:** enclosure/loop・可動体・描画禁止ゾーンは Ch1 で使わない（§2 非採用表）。
