# ギャップ調査 #4: 市場投入戦略（GTM）——マーケタビリティ検証・パブリッシャー持ち込み vs 自社配信・Webポータル参入・ソフトローンチ設計

- 調査日: 2026-07-07
- 対象: 「線を引いて橋を作り車を走らせる」物理ゲーム（iOS + Android、HTML5ビルドあり）
- 前提: 個人〜少人数開発、市場KPIデータ皆無の状態から「商用レベル」を目指す
- 調査方法: WebSearch 13回 + 一次ソース精読（Supersonic / Kwalee / Voodoo / Poki / CrazyGames / Google 公式ドキュメント等）9件

---

## 0. エグゼクティブサマリー

1. **CPIテストは「ゲーム完成前」にやるのが業界の定石**。Kwalee の公式ガイドによれば CPIテスト用ビルドは「1〜2レベルで十分、未完成・未研磨でよい」。Facebook で動画4〜8本、1広告あたり日額 $15（Android）〜$20（iOS）、6日間、broad targeting が標準プロトコル。合否ラインは **CPI < $0.25 で有望、$0.35〜0.45 で要再考**（Supersonic 基準、Android/US）。（確度: 高）
2. **完全な「フェイク広告テスト」（実ビルドなしの CTR 計測）は今も可能だが、得られるのは CTR のみで CPI は測れない**。ストアポリシー上のリスクは「広告と実ゲームの乖離」に対して強化されており（Google Play 2022年8月広告ガイドライン以降）、テスト段階では「最小実ビルドを Android に置く」方式が現在の主流。（確度: 高）
3. **2026年の重大な実務制約**: 2023年11月13日以降に作成された Google Play **個人**デベロッパーアカウントは、製品版公開前に「テスター12人 × 継続14日間」のクローズドテストが必須。CPIテストの機動力を殺すため、**法人（organization）アカウントの取得か、既存アカウントの利用が事実上の前提**になる。（確度: 高）
4. **ハイパーカジュアル専業パブリッシング窓口は縮小し、ハイブリッドカジュアル前提に再編済み**（Voodoo / Rollic / Kwalee とも hybrid-first に転換）。それでも門戸は開いている: Voodoo は 2025年のコンペで「**CPI ≤ $0.20 かつ D1 ≥ 35% で $20,000 支給**」、Supersonic は有望プロトタイプに **$300 のテスト費用を負担**。合格ラインは上がり、Voodoo のハイブリッド基準は **D1 45% / D7 15%（パズルは20%）/ D30 10%**。（確度: 高）
5. **レベシェアの相場**: ヒット時 50/50 のプロフィットシェアが「公正」とされる基準。前払い（MG）付きだと実質 70/30（パブ有利）に寄る。プロトタイプ支援金は $6,000 程度＋純利益50%〜という例も。自社配信は理論上マージン2倍だが、UA運転資金と運用ノウハウが必要。（確度: 中〜高）
6. **Webポータルは本作にとって最良の「ゼロコスト需要検証チャネル」**。Poki は開発者オンボーディングが5段階で構造化されており、**Web Fit Test（約10,000人の実プレイヤーに3〜5日露出、0〜5点スコアリング、3点以上で合格）**という事実上の無料マーケタビリティテストを提供。収益分配は Poki 経由トラフィックで 50/50。CrazyGames は Basic Launch（審査軽め）→ Full Launch の2段階で、レベシェアは**広告 60/40・IAP 70/30（開発者取り分、リクープ後）**。（確度: 高）
7. **YouTube Playables は 2026年7月時点でも Private Preview（招待制）**。興味フォーム経由の申請のみ可能で、一般開放されていない。Poki の Web Exclusive 契約は YouTube Playables / Discord への展開を5年間ブロックする点に注意。（確度: 高）
8. **ソフトローンチの標準設計**: フィリピン（CPI < $0.5、英語話者92.6%、Android中心）で技術・UX検証 → 北欧/UK で収益化検証。リテンション判定には **2,000〜5,000インストール**が必要。ゲート値は D1 30%（最低ライン）→ 35-45%（有望）、D7 15-20%、D30 10%。（確度: 高）
9. **本作の推奨GTMシーケンス**: ①Web（CrazyGames Basic → Poki 申請）でプレイタイム/継続の無料検証 → ②Android 最小ビルドで Facebook CPIテスト（$300〜500、6日）→ ③結果次第でパブリッシャー持ち込み（CPI ≤ $0.30 かつ D1 ≥ 35%）or 自社ソフトローンチ（PH → Nordics、4〜8週）→ ④グローバル。各段階に kill 基準を設定（本文 §5）。（確度: 提案）

## 1. マーケタビリティ検証（CPIテスト）の実務

### 1.1 テストの種類と使い分け

| テスト種別 | 必要なもの | 測れるもの | 費用感 | 位置づけ |
|---|---|---|---|---|
| CTR/コンセプトテスト | 静止画 or 動画のみ（実ビルド不要） | CTR（興味の有無） | $100〜300 | 本開発前の「テーマ・アート選定」 |
| CPIテスト | 最小ビルド（**1〜2レベルで可**）＋ストア掲載 | CPI / IPM / CTR | $300〜500 | コア機構のマーケタビリティ判定 |
| リテンションテスト | 充実したプロトタイプ（目安10レベル〜） | D1 / D3 / プレイタイム | $500〜1,000+ | パブ持ち込み・SL前の最終関門 |

- Kwalee 公式: 「CPIテストは1〜2レベルだけでよく、フル機能・研磨は不要。見た目の広い訴求力と気持ちいい瞬間だけに集中せよ」。リテンション計測には作り込んだプロトタイプが必要（確度: 高。出典: Kwalee ブログ）。
- CPIテストは CTR テストより信頼できる、というのが業界コンセンサス（Lancaric、Kwalee とも同旨）。CTR が高くてもストアページで離脱すれば CPI は悪化する。
- 追加で見るべき指標は **IPM（1,000インプレッションあたりインストール数）**。ハイパーカジュアルの中央値は 8.1（2023）→ **8.86（2024）**（確度: 中。出典: Segwise ベンチマーク集計）。

### 1.2 Facebook CPIテストの標準手順（Supersonic 公式プロトコル）

1. **クリエイティブ**: 動画 **4〜8本**（同時に10本超はNG＝オーディエンスが分散する）。プロトタイプ段階では編集・SEは最小限、**コアゲームプレイをそのまま**見せる。
2. **動画尺**: 最大30秒。**最初の3〜5秒に最も面白い瞬間**を置く（5秒超待たせるとフックに失敗し CPI が上がる）。
3. **予算**: 1広告あたり日額 **Android $15 / iOS $20**。
4. **ターゲティング**: broad（年齢は広く、Facebook デフォルト配置・自動オーディエンス）。狭めると結果が歪む。
5. **期間**: **6日間**（Facebook の学習フェーズが3〜4日で、CPI が安定するのを待つ）。
6. **ジオ**: US が標準（ZeptoLab は US iOS で1〜5日運用の例）。

**予算 $300〜1,000 での設計**（本作向けの試算）:
- 動画4本 × $15/日 × 6日 = **$360（Android/US）** が最小の「クリーンな1回分」。10コンセプト規模のバッチなら $300〜500 が相場（AdDogs）。
- $1,000 あれば「1回目（4本）→ 負け筋を切って勝ち筋の変奏で2回目（4本）」の**2イテレーション**が可能。
- 統計的判断の目安: 差が小さいクリエイティブ同士の比較ほど必要サンプルが増える（Lancaric）。実務上は各広告 5,000〜10,000 インプレッション・数十インストール規模で CPI の傾向は読める（確度: 中）。

### 1.3 合否閾値（2025-2026年時点、Android/US 基準）

| CPI | 判定 | アクション |
|---|---|---|
| **< $0.20** | 極めて有望 | Voodoo コンペの賞金ライン（＋D1 35%）。即パブ持ち込み級 |
| **< $0.25** | 強い | 開発続行、リテンションビルドへ |
| $0.25〜0.35 | 可能性あり | CPI改善と開発を並行 |
| $0.35〜0.45 | 疑わしい | 開発を止めて CPI 改善のみ。改善しなければ kill |
| > $0.45〜0.50 | 不合格 | コンセプト再考 or kill |

- Supersonic: 「CPI $0.30 未満を狙え」「$0.35〜0.45 はコンセプトのマーケタビリティ自体が疑問」（確度: 高）。
- ジャンル別例: ランナー系は「CPI < $0.45 かつ D1 > 30%」で報奨対象という基準例あり（ironSource）。
- 市場全体では、ハイパーカジュアルの CPI は $0.20〜0.80 のレンジで、Meta/TikTok とも前年比 15〜20% 上昇中（Admiral Media, 2025）。**閾値は年々シビアになる**方向。
- 注意: コンセプトテストの予測 CPI と実運用 CPI は **2〜2.5倍** 乖離する（Lancaric）。テストで $0.30 なら実戦は $0.6-0.75 になりうる前提で LTV 設計する。

### 1.4 「ゲーム本体なし」フェイクテストの現在（2026年）

- **CTR-only テストは今も実行可能**: 実ビルドなしで動画/静止画広告を出し、リンク先をランディングページ（メール登録計測）や類似ゲームのストアページにする方式。Lancaric の古典的手法では「プレースホルダーのテストストアページ＋スクリーンショット5枚＋レビュー数件」まで作り込む例が紹介されているが、**偽レビューの作成は現在のストア規約違反**であり推奨しない（確度: 高）。
- **政策リスクの現状**: Google Play は2022年8月の広告ガイドライン以降「アプリの実態を正確に反映しない広告」を明示的に禁止、違反はアプリの停止・削除に繋がる。Apple も同様の方向。ただしこれは「**公開アプリの広告が実ゲームと乖離している場合**」の規制であり、**ビルドを一切公開しないCTRテスト自体を直接罰する仕組みはない**（広告プラットフォーム側のポリシー＝リンク先の整合性・誤誘導禁止には注意）。（確度: 中〜高）
- 業界の現実: フェイク広告はインストール率を大きく引き上げる（CPI 40〜60%減の分析例）が、リテンション・LTVへのペナルティが年々可視化されており、**計測目的以外での常用は非推奨**（Segwise / Udonis / admapix）。
- **実務上の結論**: 2026年に個人開発者が取るべきは、(a) 本開発前 = 動画のみのCTRテスト（リンク先は事前登録LPか既存の自作ストアページ）、(b) 縦切りスライス完成後 = **Android に最小実ビルドを置いた正規CPIテスト**、の2段構え。フェイク（実装しない誇張ゲームプレイ）をテスト動画に使うと、勝っても本実装とのギャップで数字が再現しない。**テストした機構＝実装する機構**にするのが鉄則。

### 1.5 Google Play「12テスター×14日」問題（2026年の落とし穴）

- 2023年11月13日以降に作成した**個人**アカウントは、製品版公開前に「12人のテスターが14日間継続オプトイン」のクローズドテストが必須。2025〜2026年前半も要件変更なし（12人・14日のまま）。**組織（法人）アカウントと旧個人アカウントは免除**。（確度: 高。出典: Google Play Console ヘルプ）
- 影響: 「思い立って3日後にCPIテスト」ができない。対策は、①D-U-N-S を取得して組織アカウントで登録、②テスターコミュニティ（Testers Community 等）の利用を織り込み**最初のビルドを14日以上前倒しでアップ**、③iOS（TestFlight→審査）併用。CPIテストを Android 中心で回す定石と衝突するため、**プロジェクト初週にストアアカウント問題を解決しておくこと**を強く推奨。

### 1.6 TikTok でのテスト

- TikTok のアプリCPIは平均 $0.50〜1.80 とゲームでは低め。ただし広告マネージャの最低予算（キャンペーン日額 $50、広告セット日額 $20 が目安）と学習期間の要件から、**$300 クラスの少額テストは Facebook より非効率**（確度: 中）。
- 有効な使い方: Facebook で勝ち筋が見えた後のセカンドオピニオン、および**オーガニックの TikTok/Shorts 投稿でバイラル反応を無料観測**（drawing/physics 系は相性が良い）。Spark Ads はエンゲージメント率が高く CPA が 25〜40% 低い傾向だが、これはスケール段階の話。

## 2. パブリッシャー持ち込み（2025-2026年の現状）

### 2.1 業界構造の変化

- 「ハイパーカジュアルは死んだのではなく再編された」: 月間DL数は 11〜13億で横ばいのまま、**収益は約80%成長**（AppMagic 2026年展望）。収益の重心は「少し長く遊ばせ、賢く収益化する」ハイブリッドカジュアルへ移動。
- Voodoo / Rollic / Kwalee とも **hybrid-first にパイプラインを再構築済み**。Kwalee は公式に「Hypercasual isn't dead, it's just overworked」としてハイブリッド転換を宣言（PocketGamer.biz）。
- 受け入れジャンル: ハイブリッドは Merge / Match-3 / Action / FPS まで広い。**物理×描画（draw mechanic）はハイパーカジュアルの実績ジャンル**（Draw Climber は Voodoo 発のヒットで、現在も CrazyGames 等のWebポータルで現役）。ただし2026年の持ち込みでは「コア機構＋薄いメタ（アップグレード/収集/進行）」まで見せられると通りやすい。（確度: 中）
- 規模感の現実: Rollic は**年間1,000本超のプロトタイプを kill し、ソフトローンチに進むのは約40本**（合格率 ~4%）。持ち込みは「数打つ前提」のゲームであることを認識する。

### 2.2 主要パブリッシャー別の提出プログラム（2025-2026）

| パブリッシャー | 提出窓口 | テスト費用負担 | 公表されている合格ライン/基準 | 備考 |
|---|---|---|---|---|
| **Voodoo** | voodoo.io/publishing（オンライン提出。既発売ゲームの持ち込み枠もあり） | コンペではVoodooのプラットフォームでテスト実行可＋Publishing/Marketingチームの支援 | 2025年コンペ: **CPI ≤ $0.20 & D1 ≥ 35% → $20,000**。ハイブリッド本採用: **D1 45% / D7 15%（パズル20%）/ D30 10%（コアゲームプレイのみで）** | 「test fast, kill fast, iterate」。外部スタジオ発が リリースの~75% |
| **Supersonic (Unity)** | supersonic.com/publishing | **有望ゲーム提出後に $300 のCPIテスト費を支給** | CPI < $0.30 目標、**D1 38%（Android）がベンチマーク**、3大KPI = CPI / Retention / APPU | Facebook CPIテストの手順書を公開（§1.2） |
| **Kwalee** | Publishing Portal（mobile / PC&Console 両方） | 記載なし（自走テストのガイドを公開） | CPIビルドは1〜2レベルで可。リテンション計測はフルプロトタイプ | 提出しても契約義務なし（書面合意時のみ成立） |
| **CrazyLabs** | **CLIK Dashboard**（セルフサーブ型テスト基盤） | ダッシュボード上でマーケタビリティテストを実行、CPI/D1/D3/CVR/CTR をリアルタイム表示 | 過去チャレンジ例: 合格で**純収益55%シェア（1年）** | ESP（Early Submission Program）契約前でもテスト可 |
| **Rollic (Take-Two)** | 「Publishing Community」への interview 制 | 記載なし | 非公開。年間1,000+プロトタイプkill、~40本がSL進出 | テンプレート駆動のハイブリッドで2025年Q3トップ4独占 |

（確度: 各社公式・業界メディアの一次情報ベースで高。ただし数値は時期・キャンペーンで変動する）

### 2.3 契約条件（レベシェア）の相場観

- **50/50 プロフィットシェア**（UA費控除後の利益折半）が「公正な標準」とされる（Jon Hook / Ducky の公表例も 50/50）。
- **前払い・MG付きの契約は実質 70/30（パブ有利）に収斂**しがち。また「パブが投資回収するまで 0/100」という条件も増加傾向（Rami Ismail の解説）。
- プロトタイプ段階の支援例: **最大 $6,000 の開発支援金＋成果に応じ純利益50%以上**（Espresso Publishing / MyGames系の公表例。MyGames は最大60%を開発者に）。
- 交渉の含意: 少人数開発では「テスト費用の肩代わり（Supersonic の $300、CLIK の無償テスト枠）」だけでも持ち込む価値がある。**データ（CPI/D1）を自前で持っていれば交渉力が跳ね上がる**。

### 2.4 パブリッシャー vs 自社配信の損益分岐

| 観点 | パブリッシャー | 自社配信 |
|---|---|---|
| 取り分 | 利益の50%（〜60%） | 100%（＝理論上マージン2倍。Tenjin 分析） |
| UA資金 | パブが全額負担（数十万〜数百万ドル規模の出稿力） | 自己資金。広告収益の入金サイクル（60日前後）が運転資金を圧迫 |
| 専門性 | クリエイティブ量産・入札最適化・SDK/メディエーション最適化込み | 全部自分。少額でも Tenjin 等で回す方法論はある（$50k予算論） |
| 判断基準 | **CPI ≤ $0.30 & D1 ≥ 35%** を出せたら持ち込みが合理的（テスト費・UA費・スケールをパブに委ねる） | テストCPIがボーダー（$0.35-0.50）でも、Web収益やニッチ長期運用で自走する道は残る |

**結論（提案）**: 本作のように市場実績ゼロのチームは、「自社でミニテスト → 数字が出たらパブ持ち込み → 蹴られたら Web + 自社小規模運用」のハイブリッドが期待値最大。パブ持ち込みは排他契約前なら複数社並行が普通（提出＝契約義務なし、Kwalee 明記）。

## 3. Webポータル参入（Poki / CrazyGames / YouTube Playables）

### 3.1 Poki

**オンボーディング（5段階 + Bonus）**:
1. **Level 1 — Adding Your Game**: 開発者申請 → 要件を満たすビルドのアップロード（全申請に返信はしない、と明記）
2. **Level 2 — Feedback & Playtesting**: 実ユーザーのプレイ録画（**5本以上の視聴が次工程の条件**）
3. **Level 3 — Player Fit Test**: 少数プレイヤーでの適合テスト（数時間で結果）
4. **Level 4 — Web Fit Test**: **約10,000人の実プレイヤーにカテゴリページで3〜5日間露出**。指標は CTR（サムネ訴求力）/ Average Time on Page / Conversion to Play の3つを等ウェイトで、カテゴリ平均比 0〜5点スコア。**3点以上で合格**。不合格でも回数無制限で再挑戦可（**無料の Web Fit Test は月1回**という運用アナウンスあり。確度: 中）
5. **Level 5 — Final Poki Review** → リリース

**収益・契約**:
- 自分が連れてきたプレイヤーの収益は**100%開発者**、Poki がもたらしたプレイヤーは **50/50**。
- **Web Exclusive 契約はデフォルト5年**。オープンWeb独占で、**Discord / YouTube Playables への展開は不可**（Steam・コンソールは可）→ 後述のGTM設計に直結する制約。
- 非独占の選択肢は「一時金買い切り」型（マーケ支援なし）。

**技術要件（主要なもの）**:
- 初期ダウンロード **8MB 未満**目標、30FPS 以上（60FPS 目標）、16:9 レスポンシブ、モバイル/タブレットでタッチ操作強制、SDK イベント（gameplayStart は「初回入力時」発火、gameplayStop、commercialBreak / rewardedBreak）実装、報告バグの1ヶ月以内修正、各プラットフォームでユーザーの85%以上で安定動作。直近3年のミッドレンジ端末サポート。
- Unity/Godot/Phaser 等主要エンジンのガイドあり。**Poki 利用者の約40%はモバイル端末**（Similarweb 推計では32%）＝タッチUXの需要検証にそのまま使える。

**評価**: Web Fit Test は「10,000人相当の実プレイヤーによる無料マーケタビリティテスト」であり、**広告費ゼロで CTR（サムネ）・プレイ転換・滞在時間が取れる**。モバイルCPIテストの前段として最適。（確度: 高。出典: sdk.poki.com 各ページ）

### 3.2 CrazyGames

- **2段階ローンチ**: ① **Basic Launch** — SDK任意・収益化なしで即掲載（審査は軽いビジュアルQAとPEGI12準拠、外部広告禁止）。**初期DL ≤ 50MB、総容量 ≤ 250MB（SDKなしは50MB）、ファイル数 ≤ 1,500**。② **Full Launch** — プレイタイム・Conversion to gameplay・リテンションを**プラットフォーム内の他ゲームとベンチマーク比較**して選抜。SDK 必須。
- **収益分配（リクープ後）**: 広告 **開発者60% / CG40%**、IAP **開発者70% / CG30%**（IAPは招待制・Xsolla 統合）。支払いは月次・**最低€100**から。
- 対応エンジン: Unity, Godot, Defold, Phaser, PlayCanvas, Construct, Pixi.js ほか。5万プレイ到達でSDK統合の技術サポートが付く。
- **Draw Climber が現役で掲載されている**＝draw×物理系のWeb需要の傍証。（確度: 高。出典: docs.crazygames.com）

### 3.3 Web先行→モバイルの需要検証という戦略

- Poki 自身が Google AdMob のケーススタディで「**ローンチ前にゲームを巧妙にテストするプラットフォーム**」として紹介されており、Web Fit Test の仕組み自体が「本命ローンチ前の需要検証装置」。
- 2025-2026 のトレンドは「Web→モバイル」の一方通行ではなく、**クロスプラットフォーム同時運用**（Metaplay / Naavik「Web Gaming Strikes Back」）。ポータルは Poki 60M MAU、CrazyGames 35M MAU 規模。
- 個人開発の実務では「CrazyGames/Poki でプレイタイム・D1 が標準以上 → モバイル版に投資」という判断フローが低コストで機能する。ただし**Web と モバイルではユーザー層・セッション文脈が異なる**ため、Web の好成績はモバイル CPI の代替にはならない（CPI テストは別途必要）。（確度: 中）

### 3.4 YouTube Playables（2026年7月時点）

- **Developer Portal は Private Preview、参加は招待制**。一般開発者は「Playables interest form」から申請して待つのみ。収益分配・マネタイズ条件は非公開（monetization requirements ドキュメントはあるが、対象は参加済み開発者）。
- Gamedev.js Jam 2026 に公式チャレンジが出るなど開発者コミュニティへの働きかけは拡大中。技術面は WebGL/Canvas 系エンジン全般対応、**JSヒープ512MB上限**。
- **結論**: 2026年時点でGTMの当てにはできない。ただし Poki と Web Exclusive を結ぶと Playables 展開が5年間封じられるため、**Playables に将来性を感じるなら Poki 非独占（買い切り）か CrazyGames 中心を選ぶ**という分岐が生じる。（確度: 高）

## 5. 結論: 本作（draw-bridge 物理ゲーム）の推奨GTMシーケンス

### 5.0 設計思想

- 本作は HTML5 ビルドがあるため、**「Web先行の無料需要検証」→「少額CPIテスト」→「データを武器にパブ交渉 or 自社SL」**という、資金ゼロ側から積み上げる順序が最も期待値が高い。
- draw×物理はWebポータルで実績のあるジャンル（Draw Climber 等）であり、Webの初速データが出やすい。
- 逆順（先にモバイルSL）は、Google Play の 12テスター問題・広告費・iOS審査を先に踏む分だけ撤退コストが高い。

### 5.1 フェーズ計画と kill クライテリア

**Phase 0 — 準備（開発と並行、Week 0-2）**
- Google Play アカウント問題を解決（組織アカウント登録 or テスター12人×14日の前倒し開始）。
- コアループの録画から**テスト動画4本**（15-30秒、冒頭3秒に「線を引く→橋になる→車が渡る/落ちる」の快感瞬間）を制作。
- （任意・$100-200）Facebook でCTRテスト: アート方向2案 × 訴求2案。CTR 1.5%未満の案は捨てる。
- **Kill基準: なし（学習フェーズ）**

**Phase 1 — Web検証（Week 2-6、費用 ~$0）**
- CrazyGames Basic Launch（審査が軽く即掲載）＋ Poki 申請（Level 1-4）。**この時点で Poki の Web Exclusive は結ばない**（モバイル/YT Playables の自由度を保持。非独占 or 契約保留で進める）。
- 見る指標: Conversion to Play、平均プレイタイム、（Poki Web Fit Testなら）0-5スコア、CrazyGames のカテゴリベンチマーク比。
- **Go基準**: Poki Web Fit Test 3.0+/5 または CrazyGames で Full Launch 打診が来る水準（プレイタイム・再訪がカテゴリ平均超え）。
- **Pivot基準**: スコア2-3 → サムネ/序盤3レベル/操作感を改修して月1の無料再テストで2回まで再挑戦。
- **Kill基準**: 改修2回後もスコア < 2.5 かつプレイタイムがカテゴリ平均の70%未満 → コア機構の訴求力不足。モバイル投資を中止し、Web買い切り（Poki 非独占ライセンス）で回収して次作へ。

**Phase 2 — モバイルCPIテスト（Week 6-8、費用 $360-800）**
- Android 最小ビルド（3-10レベル、GameAnalytics 等の計測入り）を Google Play 公開。
- Facebook: 動画4-6本 × $15/日 × 6日、US、broad。TikTok は原則スキップ（予算効率が悪い）。
- **Go基準（パブ持ち込みルート）**: 最良動画で **CPI < $0.25**、かつテスト流入ユーザーの **D1 ≥ 35%**。
- **条件付き続行**: CPI $0.25-0.35 → クリエイティブ改修＋序盤改善で1回だけ再テスト（+$360）。
- **Kill基準**: 2回のテストで CPI > $0.40 が続く → モバイル有料UAは成立しない。Webルート（Phase 1 の収益化）に一本化。

**Phase 3A — パブリッシャー持ち込み（CPI/D1がGo基準を満たした場合）**
- Supersonic（$300テスト費支給）と CrazyLabs CLIK（セルフサーブテスト）を並行提出 → Voodoo / Kwalee にもデータ添付で提出（提出は契約義務なし）。
- 交渉ライン: プロフィットシェア50/50未満（開発者側 < 50%）で前払いもないオファーは原則断る。
- **Kill基準（このルートの）**: 3社以上に提出して全て不合格、かつ各社フィードバックが「機構の天井」を指す場合 → Phase 3B に切替えず Web運用＋次作へ。

**Phase 3B — 自社ソフトローンチ（パブ不成立だが数字がボーダーの場合）**
- PH で技術・リテンション検証（2-4週、2,000-5,000インストール、$600-1,500）→ 北欧/UK で収益化検証（2-4週）。
- コンテンツ最低ライン: SL開始時 30-40レベル＋失敗の面白さ（物理の暴発）を活かしたリプレイ性。アップデートごとに30-40レベル追加が業界の目安。
- **Goゲート（グローバル）**: D1 ≥ 30%、D7 ≥ 10%、予測LTV ≥ CPI×1.3、クラッシュ率 < 1%。
- **Kill基準**: 8週間・$3,000 投下時点で D7 < 8% または LTV/CPI < 0.8 → グローバル有料UAは中止し、オーガニック＋Web運用に縮退。

**Phase 4 — グローバルローンチ**
- パブ経由ならパブ主導。自社なら Meta 中心 → AppLovin/Unity/TikTok へ拡張（4チャネル体制が定石）。
- Web は継続運用（Poki と独占契約するのはモバイル戦略が確定した後。YT Playables の一般開放をウォッチ）。

### 5.2 縦切りスライス・メタ実装への含意

- **CPIテストに必要なのは1-2レベル**（Kwalee）→ 縦切りスライスは「1面の完璧な気持ちよさ＋広告映えする物理リアクション」を最優先。メタ（アップグレード、スキン、進行マップ）は **Phase 2 通過後（リテンションビルド段階）まで後回しでよい**。
- ただし2026年のパブ基準はハイブリッド前提（D7 15%+）なので、**メタの「設計」だけは先に用意**し（実装はしない）、持ち込み資料に含めると通過率が上がる。
- Web版は Poki SDK のイベント設計（gameplayStart/Stop、rewardedBreak）を最初から抽象化しておくと、CrazyGames SDK / YT Playables SDK への差し替えが容易。
